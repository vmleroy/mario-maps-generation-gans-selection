# Super Mario Bros. Map Generation using GANs and CMA-ES

This repository contains the code for a project that uses a Generative Adversarial Network (GAN) to learn and generate new levels for the classic game Super Mario Bros. The generated levels can be further optimized using the Covariance Matrix Adaptation Evolution Strategy (CMA-ES) to create levels with specific characteristics.

**Note:** This project was used for a *Trabalho de Conclusão de Curso* (TCC) at the *Centro Federal de Educação Tecnológica de Minas Gerais* (CEFET-MG) and was also the basis for the paper "2D Map Generation For Games Using Generative Adversarial Network," presented at the *Simpósio Brasileiro de Jogos e Entretenimento Digital* (SBGames).

## Overview

This project combines deep learning and evolutionary algorithms to generate novel and playable Super Mario Bros. levels. The core of the project is a GAN trained on existing Mario levels from the Video Game Level Corpus. This GAN learns the underlying patterns and structures of what makes a Mario level.

Once the GAN is trained, it can generate new level segments from a latent vector. To find the best and most interesting levels within the vast latent space of the GAN, this project employs CMA-ES. This evolutionary algorithm can optimize the latent vectors to produce levels that meet certain criteria, such as specific tile distributions or that are playable by an A\* agent.

## Features

  * **Generative Adversarial Network (GAN):** A PyTorch-based GAN that can be trained to generate Super Mario Bros. level segments.
  * **Level Generation:** Generate new, unique Super Mario Bros. levels from random latent vectors.
  * **Evolutionary Optimization:** Utilizes CMA-ES to evolve and select the best-generated levels based on desired features.
  * **Playable Levels:** The generated levels can be played by a human or an AI agent.
  * **Modular Codebase:** The project is structured with Python code for the GAN and Java code for the Mario AI framework and CMA-ES.

## How It Works

The project is divided into two main components:

1.  **The GAN (in Python/PyTorch):**

      * The GAN is trained on a dataset of Super Mario Bros. levels.
      * The generator network takes a latent vector (a point in a high-dimensional space) as input and outputs a 2D representation of a level segment.
      * The discriminator network is trained to distinguish between real level segments and the ones generated by the generator.
      * Through this adversarial process, the generator learns to produce realistic and coherent level segments.

2.  **The Mario AI Framework and CMA-ES (in Java):**

      * This component uses the trained GAN to generate levels and then evaluates them.
      * CMA-ES is used to search the latent space of the GAN for vectors that produce "good" levels. "Good" can be defined by various fitness functions, including:
          * **Static Features:** Optimizing for a certain distribution of tiles (e.g., more enemies, fewer blocks).
          * **Agent Performance:** Evaluating how well an A\* agent can play the level.
      * The generated levels can be stitched together to create longer, more complex levels.

## Getting Started

### Prerequisites

  * Python 3
  * PyTorch
  * Java
  * NVIDIA GPU + CUDA (recommended for training the GAN)

### Installation

1.  **Clone the repository:**

    ```bash
    git clone https://github.com/vmleroy/mario-maps-generation-gans-selection.git
    cd mario-maps-generation-gans-selection
    ```

2.  **Install Python dependencies:**

    ```bash
    pip install -r requirements.txt
    ```

## Usage

### Training the GAN

An already trained model is provided in the `pytorch/` directory (`netG_epoch_5000.pth`). However, if you want to train the GAN from scratch, you can run:

```bash
python main.py
```

### Generating Levels

You can generate and view random level segments using the provided Java classes.

  * To generate and view a level from a random latent vector, use `viewer.MarioRandomLevelViewer`.
  * To generate and view a level from a specific latent vector, use `viewer.MarioLevelViewer` with a JSON array of length 32 as a command-line argument.
  * To play a generated level, use `viewer.MarioLevelPlayer`.

### Evolving Levels

To evolve levels using CMA-ES based on agent performance, you can use the `cmatest.CMAMarioSolver` Java class. This will use an A\* agent to evaluate the levels and guide the evolutionary search for better levels.
